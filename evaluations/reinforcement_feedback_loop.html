Reinforcement And Adaptive Weight Loop

This page isolates the part of the system that learns from retrieval outcomes. Attribution is cached at retrieval time, implicit or explicit signals arrive later, Hebbian reinforcement updates memory salience and graph coactivation, and the hybrid search learner shifts BM25, vector, and graph blend weights toward what actually helped.

Exact Anchors

Loop State Surfaces

The reinforcement loop changes more than a single score. It alters multiple persistent surfaces whose interaction determines future recall quality.

Attribution Cache

Retrieval stores per-memory signal contributions in last_signal_attributions so delayed supervision can still train on the actual ranking context that produced the answer.

Feedback Store

Pending response state captures followup text, context embedding, and tool usage so implicit feedback can be classified after the user responds.

Graph Coactivation

Helpful sets of memories are pushed through GraphMemory.record_memory_coactivation, which means reinforcement strengthens structure, not only scalar memory weights.

Hybrid Blend Learner

Average BM25, vector, and graph contributions feed update_weights_from_feedback so future retrieval uses a gradually learned mix instead of fixed static fusion.
