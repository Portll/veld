# Glossary

Terms used across veld's documentation and codebase.

---

**Activation level**
A scoring signal (signal 17, 3% weight) tracking how recently a memory participated in a retrieval or consolidation cycle. High activation boosts rank; decaying activation reflects a memory moving toward the background.

**Alignment (cross-embedder)**
A Procrustes + Ridge linear map from one embedder's vector space to another (e.g., MiniLM 384d → Nomic 768d). Alignment makes dual-embedder max-score merging meaningful by putting both embedders' outputs in a comparable space. See `src/bin/alignment-fit`.

**Anchor**
An agent-pinned memory that resists decay. The `POST /api/anchor` endpoint sets the anchor flag. Useful for long-lived facts (e.g., "user prefers X over Y") that should never be archived.

**Arousal**
A facet encoding the emotional intensity of a memory event. High-arousal memories (urgent, surprising, emotionally significant) are boosted in retrieval.

**BM25**
Best Match 25 — a probabilistic keyword-based ranking function. Veld uses Tantivy's BM25 implementation as the full-text retrieval layer, combined with vector search and graph spreading activation.

**Calibrated confidence**
A Bayesian α/β pair per memory encoding how confident the system is in the memory's factual accuracy. New memories start at a prior; observations (user confirmations, contradictions) update the posterior. Retrieval gates at 0.85–1.0 — low-confidence memories are surfaced for verification but do not pollute factual recall.

**Context block**
A Letta-style mutable agent state entry (`key`, `content`, `max_tokens`, `version`). Unlike memories, context blocks are persistently editable — the agent can overwrite them. Distinct from the memory store; useful for working notes, current task, and agent identity.

**Cross-encoder reranker**
A more expensive bi-directional text comparison model that scores query-memory pairs jointly (rather than independently as bi-encoders do). Veld runs the cross-encoder over the top-20 candidates from the initial retrieval pass and blends the score at 18%.

**Decay**
The reduction in a memory's effective importance over time. Veld uses multi-time-scale decay (`src/decay.rs`) with Fourier-learned decay scales per memory type. Anchor and Hebbian strengthening counteract decay.

**Derived memory**
A memory generated by consolidation from other memories, not directly from agent input. Examples: fact extraction summaries, merged near-duplicates.

**Dual-embedder competition**
Running two embedding models in parallel (MiniLM 384d as primary, Nomic 768d as secondary) and taking the max similarity score across both. Implemented by `CompetitiveEmbedder` using `Arc<dyn Embedder>`.

**EngramBinding**
A facet linking a memory to a specific learning episode or engram cluster. Supports episodic coherence scoring (signal 9).

**Episode coherence**
Scoring signal 9 (8% weight). Memories that were encoded in the same session or share EngramBinding facets get a coherence boost when retrieved together.

**External dimension scores**
Five scores pushed from Sleight's evaluation engine via `POST /api/sleight/dimensions`: density, coherence, closure, confidence, isotropy. These modulate retrieval rank as signal 20 when fresh. Allows Sleight to exert fine-grained influence on which memories surface.

**Facet**
A structured annotation attached to a memory. Facets include `Who`, `What`, `When`, `Where`, `Why`, `RecordKind`, `ContentKind`, `CausalLink`, `EngramBinding`, `Prediction`, `AgentRef`, `Place`. Used for graph-based retrieval and the wiki-render page taxonomy.

**Feedback momentum**
Scoring signal 11 — an EMA (exponential moving average) over user-provided reinforcement signals (`POST /api/reinforce`). A memory that users consistently upvote accrues positive momentum; consistently downvoted memories decay faster.

**Gap topology**
A subsystem (`src/memory/gap_topology/`) that identifies knowledge gaps in the graph. Uses Voronoi decomposition and persistent homology to find regions of memory space with sparse coverage. Surfaces gaps as hints for what the agent should learn next.

**Hebbian learning**
"Neurons that fire together, wire together." In veld, memories that are retrieved together in the same context accumulate stronger graph edges. These edges feed graph spreading activation and boost the `graph_strength` signal (8%). See `src/memory/graph_retrieval.rs`.

**HNSW**
Hierarchical Navigable Small World — an approximate nearest-neighbour index for high-dimensional vectors. Veld's default vector retrieval engine (`src/vector_db/`).

**Intent log**
An append-only, event-sourced journal (`src/intent_log/`) that records every agent action (remember, recall, reinforce, etc.) as a typed `IntentPayload` with bincode encoding. The log is the ground truth for agent behaviour, independent of the memory store state.

**Interference detection**
Layer 4.92 in the retrieval pipeline. Pairs of semantically opposing memories (e.g., "X is true" and "X is false") are detected; the older one is demoted to prevent contradictory results surfacing together.

**Memory tier**
The lifecycle stage of a memory: `Working → Session → LongTerm → Archive`. Promoted by importance × access frequency × age. Agent can direct tier moves via `POST /api/memory/tier`. Working and Session tiers are brute-force scanned; LongTerm and Archive use HNSW.

**MIF (Memory Interchange Format)**
Import/export adapters (`src/mif/`) for Mem0, Veld native, markdown, and generic formats. Includes PII handling for export.

**Page contract**
A CLAUDE.md-defined schema that specifies what fields a rendered wiki "page" must contain. Used by the LLM-wiki render module to validate generated page output before it is committed. See the [schema/page-contract.md](schema/page-contract.md) page (generated from CLAUDE.md).

**Procrustes alignment**
An orthogonal linear transformation that rotates one matrix to best fit another in the least-squares sense. Used in the cross-embedder alignment pipeline to align MiniLM's 384d output to Nomic's 768d space.

**Prospective memory**
A memory of type `ExperienceType::Intention` — a future-directed reminder. Filtered from normal recall; surfaces via `/api/reminders` or when spreading activation reaches it during a session.

**Ridge regression**
A regularised linear regression (L2 penalty) used in the alignment pipeline to learn a linear map between embedder spaces when Procrustes is insufficient (non-square case, or when a bias term is needed).

**Semantic proximity**
How similar the content of two memories is in embedding space. Used by interference detection and by deduplication during consolidation.

**Signal attribution**
Per-retrieval-per-memory tracking of which of the 20 scoring signals contributed to a memory's rank. Enables adaptive weight learning: if a signal consistently predicts relevance, its weight can be increased; if it consistently misleads, it can be discounted.

**Source tier**
A multiplier on source credibility (signal 10) based on the type of source: `User` (highest), `Agent`, `Document`, `Unknown` (lowest).

**Spreading activation**
A graph-based retrieval technique where similarity scores propagate along Hebbian edges. A high-scoring memory activates its neighbours, which activates their neighbours, gradually surfacing related context that a pure vector search would miss.

**Vamana / DiskANN**
An approximate nearest-neighbour index optimised for disk-resident data. Available in `src/vector_db/` alongside HNSW. Used for LongTerm-tier retrieval when the 768d Nomic secondary index is enabled.

**Wavelet session**
A session-segmentation technique (`src/memory/wavelet_sessions.rs`) that uses wavelet analysis over the timeline of memory access patterns to detect session boundaries automatically, without relying on explicit "session start/end" signals.

**Bincode**
A compact binary serialization format. Used for `IntentPayload` encoding in the intent log — chosen over JSON for size and parse speed.

**ContentKind**
A facet describing the *form* of a memory's content: `Code`, `Prose`, `StructuredData`, etc. Distinct from `RecordKind` (which describes the *role* of the memory).

**External dimension scores**
See full entry above; pushed by Sleight via `POST /api/sleight/dimensions`. Carry `is_stale()` for freshness checking.

**Fortress**
A distribution-build feature flag. Strips tracing, replaces error messages with codes, installs a custom panic handler, and adds anti-debug + integrity checks. For shipping veld to untrusted environments.

**Hosaka**
The internal codename for the multi-tenant collective-learning subsystem. Multiple veld tenants share a "collective store" with explicit access control. Behind the `multi-tenant` feature flag.

**IntentPayload**
The typed enum variant inside the intent log. Each agent action (remember/recall/reinforce/anchor/forget/tier_move/etc.) has a corresponding variant. bincode-encoded for storage.

**Letta-style mutable state**
A pattern from the Letta agent framework: persistently editable agent state distinct from immutable memories. Veld implements this as `ContextBlock` (see above) — agents can read, write, and version these without going through the consolidation pipeline.

**MIF (Memory Interchange Format)**
A versioned export/import format ([`src/mif/`](https://github.com/Portll/veld/tree/main/src/mif)) with adapters for Mem0, Veld native, markdown, and a generic adapter. Supports PII handling on export.

**ONNX**
Open Neural Network Exchange — the format veld uses for embedded embedder model files. MiniLM-L6-v2 is shipped as ONNX; loaded at runtime via `ort` crate.

**Persistent homology**
A topological method used by gap topology to detect connectedness gaps in the knowledge graph. Identifies regions of memory space with sparse coverage that look like "holes" topologically.

**PQ (Product Quantization)**
A vector compression codec. Used in `src/vector_db/pq.rs` to reduce embedding storage size while preserving approximate nearest-neighbour quality. Can be layered on top of HNSW or Vamana.

**RecordKind**
A facet describing the *role* of a memory: `Episodic`, `Semantic`, `Procedural`, etc. Used by the wiki-render pipeline as part of the page-type taxonomy.

**ScoringSignals**
The 20-field struct in `src/memory/types.rs` that holds the inputs to the composite ranking score. Populated during retrieval by extracting signals from each candidate memory.

**Sleight**
An external evaluation engine that pushes topological-health scores into veld via `POST /api/sleight/dimensions`. Sleight computes the scores using gap analysis + Voronoi decomposition on veld's graph API; veld consumes them as the 20th retrieval signal.

**SPANN**
A vector ANN algorithm — Space Partition Approximate Nearest Neighbour. Two-level partitioned index for very large vector collections. Available in `src/vector_db/spann.rs` alongside HNSW.

**Tantivy**
A Rust full-text search library, similar to Lucene. Veld uses Tantivy's BM25 implementation as the keyword-match layer in the hybrid retrieval pipeline.

**Tokio task / `tokio::task::spawn`**
The async-runtime primitive veld uses for background work. Consolidation runs as a detached task so it survives HTTP request cancellation. New write paths that need to outlive the request should follow the same pattern (see `consolidation.rs` `CONSOLIDATION_LOCKS`).

**Voronoi decomposition**
A geometric tiling where each region contains the points closest to a chosen seed. Veld uses Voronoi over the knowledge graph to identify entity neighbourhoods — regions with sparse memory coverage become candidate "gaps" the agent should fill.

**Zenoh**
Eclipse Foundation's protocol for distributed edge systems. ROS2-native pub/sub with low overhead. Veld's `zenoh` feature flag enables Zenoh transport for multi-node deployments (`src/zenoh_transport/`).
