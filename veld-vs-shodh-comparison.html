shodh-memory → veld  ·  Lineage & Feature Comparison

Generated 2026-05-29 · Comparative inventory aligning the fork ancestor against the downstream evolution. Every assertion in this document is sourced to a file path (and where useful, a line number or quoted excerpt) inside one of the two repositories.

Lineage statement (per user): shodh-memory is the original base; veld is the downstream fork/rebrand. Both share the same overall architecture (Rust core + TypeScript MCP wrapper + Python bindings + ONNX MiniLM + RocksDB + Vamana + Tantivy + Hebbian graph). The diff is in (a) what shodh shipped and veld kept, (b) what veld extended after the fork, (c) what shodh-side features did not propagate forward, and (d) renames (the most visible: shodh binary → veld binary, @shodh/memory-mcp → @veld/memory-mcp, SHODH_* env → VELD_* env).
Erratum (2026-05-29, post-publication). The first version of this report incorrectly tagged four items as SHODH-ONLY: WebSocket streaming ingest, Prometheus /metrics, embedded graph viewer, and the Telegram/Todoist/WhatsApp connector packages. All four are in fact already present in veld and have been re-tagged below:
  • src/streaming.rs (1577 lines) carries the same WebSocket /api/stream architecture → INHERITED
  • src/metrics.rs (690 lines, shodh_* metric prefix renamed to veld_*) + /metrics route in router.rs:122-124 with VELD_METRICS_PUBLIC conditional → RENAMED
  • src/handlers/visualization.rs + graph_view.html + GET /graph/view route + /api/visualization/* family + /api/brain/{user_id} → EXTENDED
  • veld-telegram/ (903 LOC, identical file set), veld-todoist/ (562 LOC, shodh.ts→veld.ts), veld-whatsapp/ (998 LOC, ahead of shodh by ~100 LOC in llm.ts and config.ts) → RENAMED (+ veld-whatsapp extended)
Root cause: the veld inventory was built primarily from CLAUDE.md's enumerated tool/route table, which does not list every Rust module by name (e.g. streaming.rs, metrics.rs, visualization.rs are not enumerated in CLAUDE.md). The independent file-tree pass that would have caught the false negatives wasn't done in the first inventory. Corrections have been applied to §3, §12, §19, §25.
